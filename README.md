# FlowForge

**Self-hosted Low-code платформа автоматизации процессов с интегрированным ИИ**

Визуальный конструктор рабочих процессов, вдохновлённый n8n, написанный с нуля на Node.js, React и React Flow. Объединяйте HTTP-запросы, JavaScript-скрипты и ИИ-агентов в автоматизированные цепочки через интерфейс drag-and-drop — без зависимости от облачных сервисов.

---

## Возможности

| Функция | Описание |
|---|---|
| **Визуальный редактор** | Холст drag-and-drop на базе React Flow |
| **Дашборд воркфлоу** | Создание, переименование, удаление и переключение между процессами |
| **Импорт / Экспорт** | Сохранение воркфлоу в `.json`; загрузка готовых пресетов |
| **Серверное выполнение** | Движок обхода графа запускает всю цепочку на бэкенде |
| **Интерполяция переменных** | Обращение к выходу предыдущего узла через `{{input.field.path}}` в любом поле |
| **Узел HTTP Request** | GET / POST / PUT / DELETE с заголовками, телом и ИИ-автозаполнением |
| **Узел Code (JS)** | Выполнение произвольного JavaScript в sandbox-среде Node.js `vm`; доступны `input`, `axios`, `console` |
| **Узел AI Agent** | Системный промпт + пользовательский промпт (с переменными `{{input}}`), выбор модели, кнопка «Test» |
| **ИИ-конфигуратор HTTP** | Опишите запрос на русском языке → поля заполняются автоматически |
| **Триггер Webhook** | Уникальный URL для каждого воркфлоу; POST любого JSON запускает выполнение |
| **Триггер Schedule** | Планировщик на базе cron с пресетами (каждый час, каждый день…) или произвольным выражением |
| **Настройки API в интерфейсе** | Ввод ключа OpenRouter через UI — без редактирования файлов |
| **Произвольные ИИ-модели** | Любой идентификатор модели OpenRouter/OpenAI вводится прямо в узле |

## Быстрый старт (Docker)

Самый простой способ развернуть FlowForge — через Docker Compose. Не требует установки Node.js.

### Предварительные требования

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) (включает Docker Compose)

### 1. Клонировать репозиторий

```bash
git clone https://github.com/YOUR_USERNAME/n8n-project.git
cd n8n-project
```

### 2. Запустить сборку и старт

```bash
docker-compose up --build
```

Команда соберёт образы фронтенда и бэкенда, запустит оба контейнера и создаст именованный volume для хранения данных.

### 3. Открыть приложение

| URL | Описание |
|---|---|
| **http://localhost:5173** | Веб-интерфейс FlowForge |
| http://localhost:3001 | Backend API (для вебхуков и интеграций) |

### 4. Настроить API-ключ

Нажмите иконку ⚙ на дашборде, введите ключ OpenRouter и нажмите **Сохранить**. Бесплатный ключ: **https://openrouter.ai/keys**

> **Данные сохраняются** между перезапусками контейнеров благодаря Docker volume `flowforge-data`.

### Управление контейнерами

```bash
docker-compose up -d          # запуск в фоне
docker-compose down           # остановка (данные сохраняются)
docker-compose down -v        # остановка и удаление всех данных
docker-compose logs -f server # просмотр логов бэкенда
```

## Ручная установка (без Docker)

Альтернативный способ — запуск из исходников через npm.

### 1. Клонировать репозиторий

```bash
git clone https://github.com/YOUR_USERNAME/n8n-project.git
cd n8n-project
```

### 2. Настроить переменные окружения

```bash
cd server
cp .env.example .env
```

Откройте `server/.env` и укажите свой API-ключ:

```env
OPENAI_API_KEY=sk-or-v1-your-key-here
OPENAI_BASE_URL=https://openrouter.ai/api/v1
PORT=3001
```

> Ключ также можно указать позже через иконку ⚙ на дашборде — изменения применяются мгновенно.

### 3. Установить зависимости

```bash
cd ..
npm run install:all
```

Команда запускает `npm install` в корне проекта, в `server/` и `client/`.

### 4. Запустить сервер разработки

```bash
npm run dev
```

| URL | Сервис |
|---|---|
| http://localhost:5173 | Frontend (React + Vite) |
| http://localhost:3001 | Backend API (Express) |

## Интерполяция переменных

Любое текстовое поле разрешается во время выполнения на основе выхода предыдущего узла:

```
{{input.data.current_weather.temperature}}   → 15
{{input.text}}                               → "Текст от ИИ"
{{input.payload.user_id}}                    → 42
```

**Пример цепочки:**

```
Start
  → HTTP GET  https://api.open-meteo.com/…?current_weather=true
  → AI Agent  "Напиши прогноз погоды для {{input.data.current_weather.temperature}}°C"
  → HTTP POST https://api.telegram.org/bot<TOKEN>/sendMessage
              body: {"chat_id": "12345", "text": "{{input.text}}"}
```

## Справочник API

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/api/workflows` | Список всех воркфлоу (метаданные) |
| `POST` | `/api/workflows` | Создать новый воркфлоу |
| `GET` | `/api/workflows/:id` | Получить воркфлоу с узлами и рёбрами |
| `PUT` | `/api/workflows/:id` | Сохранить узлы, рёбра и название |
| `DELETE` | `/api/workflows/:id` | Удалить воркфлоу и его cron-задачи |
| `POST` | `/api/run` | Запустить воркфлоу из редактора |
| `POST` | `/webhook/:workflowId` | Внешний webhook-триггер (тело запроса → выход webhookNode) |
| `POST` | `/api/ai-agent` | Прямой вызов языковой модели |
| `POST` | `/api/ai-config` | Генерация конфигурации HTTP-узла по текстовому описанию |
| `GET` | `/api/settings` | Чтение сохранённых API-учётных данных |
| `POST` | `/api/settings` | Сохранение API-учётных данных |

## Импорт и экспорт

**Экспорт:** нажмите **Export** в тулбаре редактора — воркфлоу скачивается как `название.json`.

**Импорт:** нажмите **Import** на дашборде и выберите `.json`-файл. Новый воркфлоу создаётся автоматически.

Формат переносим: воркфлоу можно передавать коллегам или публиковать как готовые пресеты.

## Планы развития

Следующие функции запланированы к реализации в будущих версиях платформы:

- **Нативные триггеры Telegram / VK** — получение сообщений и событий напрямую от Telegram-ботов и VK Callback API без необходимости вручную отправлять HTTP-запросы. Для локальной разработки потребуется сервис туннелирования (ngrok / Cloudflare Tunnel); для production-среды — собственный сервер с публичным доменом.

- **Маркетплейс сообщества** — централизованный каталог, в котором пользователи смогут публиковать и загружать готовые пресеты воркфлоу. Пресеты распространяются как версионированные JSON-файлы с системой рейтингов и импортом в один клик, что позволяет строить экосистему многократно используемых шаблонов автоматизации.

- **Расширенные ИИ-агенты на базе LangChain** — замена простого узла с одним промптом на полноценный LangChain-агент с поддержкой вызовов инструментов (веб-поиск, интерпретатор кода), многошаговых рассуждений (chain-of-thought) и сохранения памяти между запусками воркфлоу.

- **Корпоративная панель администратора** — многопользовательская модель с ролевым разграничением прав доступа, централизованным управлением API-ключами на уровне рабочего пространства, журналами аудита и квотами на потребление ресурсов.
