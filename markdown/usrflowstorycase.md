## Кейс использования

**Профиль пользователя:** Владелец малого бизнеса Маркетолог. Не умеет программировать и читать документацию к API, но хочет автоматизировать рутину.
**Задача:** Каждое утро получать в Telegram актуальный курс Bitcoin для принятия решений.

**Путь пользователя в системе:**

1. **Инициализация:**
   - Пользователь запускает систему одной командой `docker-compose up`.
   - Заходит в веб-интерфейс, нажимает на "Шестеренку" (Settings) и безопасно вводит свой API-ключ OpenRouter. Больше к настройкам возвращаться не нужно.
2. **Настройка триггера:**
   - Пользователь создает новый Workflow.
   - Перетаскивает узел **Schedule** и выбирает настройку, подходящую для себя.
3. **Получение данных:**
   - Перетаскивает узел **HTTP Request**.
   - Вместо долгого изучения документации CoinGecko, пишет в поле "Ask AI": *"Get current Bitcoin price in USD"*.
   - Нажимает "Generate". Система автоматически прописывает правильный URL и метод GET.
4. **Обработка данных: (Code Node)**
   - Перетаскивает узел **Code Node**.
   - Пишет пару строк JS-кода для извлечения конкретной цифры из большого JSON-ответа (например, `return { price: input.data.bitcoin.usd }`).
5. **Трансформация данных данных (AI Text Node):**
   - Добавляется узел **AI Text**. 
   - В поле промпта используется интерполяция переменных: *"Напиши короткий креативный пост для соцсетей, учитывая эту температуру: {{input.data.current_weather.temperature}}°C"*.
   - Узел обращается к LLM и возвращает готовый отформатированный текст.
6. **Отправка результата:**
   - Берет готовый пресет **Telegram Message** из левой панели.
   - Вставляет свой Chat ID и пишет текст с использованием переменной из прошлого шага: `"Курс биткоина: {{input.price}}$"`.
7. **Запуск и сохранение:**
   - Нажимает "Run Workflow" для проверки (справа появляется лог успешного выполнения).
   - Нажимает "Save". Теперь сервер сам будет выполнять этот процесс каждый день в фоне.
