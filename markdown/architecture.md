## Архитектура сервиса

Сервис построен по паттерну **Client-Server** с использованием контейнеризации (Docker) для обеспечения простого Self-hosted развертывания.

**Стек технологий:**
- **Frontend:** React (Vite), React Flow, Tailwind CSS.
- **Backend:** Node.js, Express, Axios, `node-cron`.
- **Хранилище:** `lowdb` (JSON-ориентированная файловая БД).
- **Инфраструктура:** Docker, Docker Compose, Nginx.

**Ключевые архитектурные решения:**

1. **Изолированный движок исполнения (Execution Engine):**
   - В основе лежит алгоритм обхода графа. Движок последовательно выполняет узлы, передавая весь `Output` предыдущего узла как `Input` для следующего.
   - **Интерполяция данных:** Реализован парсер переменных, который "на лету" заменяет конструкции вида `{{input.data.value}}` на реальные значения перед выполнением запроса.

2. **Безопасная среда выполнения (Sandboxing):**
   - Для узла `Code Node` используется встроенный модуль Node.js `vm`. 
   - Пользовательский JavaScript-код выполняется в изолированном контексте (имеет доступ только к переменной `input` и `console`), что предотвращает крах основного сервера в случае ошибок или зловредного кода.

3. **Маршрутизация и Proxy (Nginx):**
   - В Docker-контейнере фронтенда поднят **Nginx**. Он отдает статику React-приложения, а все запросы, начинающиеся с `/api/` и `/webhook/`, прозрачно проксирует на Backend-контейнер. Это решает проблему CORS и упрощает сетевую конфигурацию.

4. **Персистентность данных (Volumes):**
   - База данных (`db.json`) вынесена в Docker Volume (`/data`). При перезапуске, обновлении или падении контейнера все Workflow, логи и API-ключи пользователей гарантированно сохраняются.

5. **Механизм планирования (Scheduler):**
   - Бэкенд инициализирует расписания (`node-cron`) при старте сервера и динамически обновляет их через API (события Save/Delete), не требуя перезагрузки всего приложения.
